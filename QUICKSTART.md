# ⚡ クイックスタートガイド

**Request 管理システム v2.0** をすぐに始めるための最短手順です。

---

## 📦 インストール（5分）

### 1. 依存パッケージをインストール

```bash
npm install
```

### 2. 環境変数ファイルを作成

```bash
cp .env.example .env
```

`.env` ファイルを開いて `SESSION_SECRET` だけ変更：

```env
SESSION_SECRET=my-super-secret-key-12345
```

その他の設定（Talknote API設定）はすでに入力済みです。

---

## 🚀 起動（1分）

### 開発モード（自動リロード）

```bash
npm run dev
```

### 本番モード

```bash
npm start
```

ブラウザで `http://localhost:3000` を開く。

---

## ⚙️ Talknote設定（必須）

アプリを使う前に、Talknote管理画面で設定が必要です。

### 開発環境の場合

1. Talknote管理画面にアクセス
2. **ネットワーク管理** > **外部公開API** を開く
3. リダイレクトURLを以下に変更：
   ```
   http://localhost:3000/api/callback
   ```
4. 保存

### 本番環境の場合

リダイレクトURLを本番URLに変更：
```
https://insp.co.jp/api/callback
```

## 🎯 使い方（3ステップ）

### ステップ1: ログイン

1. `http://localhost:3000` にアクセス
2. 「Talknoteでログイン」をクリック
3. Talknoteアカウントでログイン

### ステップ2: リクエストを作成

1. 「新規リクエスト作成」ボタンをクリック
2. フォームに入力：
   - **タイトル**: 例）「会議室予約の確認」
   - **内容**: 例）「11/15の会議室を予約したいです」
   - **期日**: 日時を選択
   - **宛先**: Talknoteユーザーを選択
   - **通知先**: Talknoteグループを選択
3. 「作成」をクリック

→ Talknoteの選択したグループに自動投稿されます！

### ステップ3: リクエストに回答

1. 受信したリクエストカードをクリック
2. 回答を選択：
   - **Yes（承認）**
   - **No（却下）**
   - **条件付きYes**（条件を入力）
3. 「回答する」をクリック

→ Talknoteに自動通知されます！

---

## 🆕 期日延長機能の使い方

### 受信者（リクエストを受けた人）

承認済みリクエストの期日を延長したい場合：

1. リクエストカードの「期日延長を申請」ボタンをクリック
2. 新しい希望期日を選択
3. 延長理由を入力（必須）
4. 「申請する」をクリック

→ 送信者に通知され、承認待ちになります。

### 送信者（リクエストを出した人）

延長申請を受け取ったら：

1. 延長申請中のリクエストカードをクリック
2. 申請内容を確認
3. 「承認する」または「却下する」をクリック

→ 承認すると期日が自動更新されます！

## 📊 ダッシュボードの見方

### タブ切り替え

- **受信したリクエスト**: 自分が受け取ったリクエスト
- **送信したリクエスト**: 自分が送ったリクエスト
- **すべて**: 全てのリクエスト

### 統計カード

- **総リクエスト数**: 自分が関わる全リクエスト
- **承認待ち**: まだ回答されていないリクエスト
- **期日内完了**: 期日までに完了したリクエスト
- **期日超過**: 期日を過ぎて完了したリクエスト
- **延長申請中**: 現在延長申請中のリクエスト

## 🔧 よくある質問

### Q: ログインできない

**A**: Talknote管理画面でリダイレクトURLが正しく設定されているか確認してください。

開発環境: `http://localhost:3000/api/callback`  
本番環境: `https://insp.co.jp/api/callback`

### Q: リクエストがTalknoteに投稿されない

**A**: Talknote API設定を確認してください。

1. クライアントIDとシークレットが正しいか
2. 外部公開APIが有効になっているか
3. 必要なスコープ（権限）が付与されているか

### Q: ファイルをアップロードできない

**A**: 以下を確認してください。

- ファイルサイズが10MB以下か
- `uploads/` ディレクトリが存在するか（自動作成されます）

### Q: 期日延長申請ボタンが表示されない

**A**: 以下の条件を満たしている必要があります。

- 受信者（リクエストを受けた側）である
- リクエストが承認済み（status = accepted）
- 現在の期日前である
- 現在延長申請中でない、または前回の延長申請が却下されている

## 🎨 デザインについて

本バージョン（v2.0）は**ミニマルデザイン**を採用しています。

### デザイン哲学

- **Less is More**: 必要最低限の要素のみ
- **余白の活用**: 情報を整理し、視認性を向上
- **モノクローム基調**: 白・黒・グレー + 青のアクセントカラー
- **フラットデザイン**: シャドウとグラデーションを排除

### カラーパレット

- メインテキスト: `#000000`（黒）
- サブテキスト: `#6b7280`（グレー）
- ボーダー: `#e5e7eb`（薄いグレー）
- アクセント: `#2563EB`（青）
- 背景: `#ffffff`（白）
